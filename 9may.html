*Graphic user Interface (GUI)*

The most widely used type of UI in use today is the Graphical User Interface (GUI). it accepts input through the keyboard, mouse abd display the output on the user,s computer screens. The action in a GUI are usually performed through direct manipulation of the graphical elemnets.

The most common elemnetsof a GUI include:

1 Window

2 Icon

3 Menu

4 Cursor pointer

*Process of user Interface Design*

1 Interface analysis.

2 Interface design.

3 Interface construction.

4 Interface implementation.

4 Interface validation.

**Interface Analysis**

-This is like studying how things are laid out and work on a website or an app.

-We look at what are these (like buttons, menus, text books)

-Where they are placed, and how they function.

-It's a something like Like taking apart a toy to see how it's made and how it works, but with website and apps instead.

**Interface Design**

-We decide how the interface will look and work based on what we learned during analysis.

-We choose colors, fonts, and shapes to make it look nice and easy to use.

-It's like a drawing a blueprint or sketch for a house beforre building it, but for a website or app.

**Interface Construction**

-this stage involves assembling all the elements needed for the interface based on the design.

-it's like putting together all the buttons, menus, and other components are in place and functioning correctly.

-Think of its as building a structure using the materials and plans provided by the design phase.

**Interface Implementation**

-Here, we actually build the interface based on the design we made.

-Develpers write the code to make everthing work as it should, like making buttons clickable and menus open when you tap them.

-it's like contructing a building according to the blueprint we draw.

**Interface Validation**

-After we've built interface, we test it to make sure it works well.

-We check if all the buttons do what they're supposed to, if the colors and fonts look good, and if it's easy for people to use

-It's similar to trying out a new gam,e to see if it's fun and works smoothly

*Principle and Attributes of User Interface Design*

**Structure Principle**

This principle is concerned with overall UI acrhitecture
The deisgn of the interface should be visually, theoretically, and lignuistically clear. (Linguistic: Pertaining to language, include structure, meaning and usage) User can easily navigate between different, parts of a websit and quickly find important information or sections.

Tips/Techniques to support the structure principle

You should group the logically connected items to communicate and separate unrelated item to achieve visual organization

Design page-specific navigation and access to information.

Keep the screen less cluttered and easier to understand

Present the flow of actions, information, responses, and visual preprations in a sensible order that is easy to remember and place in context
